What is a layout?

A layout is functionally the same as a block <div>. The argument for using layouts over blocks is to make code more readable. For example, if you want a section of your page to be beside another section, using an <aside> block is better than a <div> because when you see it, you know exactly the intention of it.
That is essentially the only reason to use a layout over a traditional block <div>.

Types of Layouts

⛶ <header>
❯ A header pane for a section.
⛶ <nav>
❯ Navigation bar is good for putting links to other pages side by side.
⛶ <section>
❯ Similar to a block <div>
⛶ <article>
❯ Similar to a <section>, but generally reserved for text only.
⛶ <details>
⛶ <summary>
❯ A nice place to summarize the content of the page.

As previously stated, all layouts are just different names for a block <div>. All layouts have the same format: 

            <layout tag>
                content
            </layout tag>
Here are some example layouts:

Header example

        <header style="background-color:green; color:white;">
            <h3 style="text-align:center;">This is an example of a header. Put a title here.</h3>
        </header>

                    <li> <a href=#nav> This is a nice place for links.</a></li>

                </ul>
            </nav>
            </body>
